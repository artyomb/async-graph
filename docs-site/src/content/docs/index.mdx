## Why AsyncGraph?

<CardGrid stagger>
	<Card title="Minimal Runtime" icon="open-book">
		Keep scheduling and persistence outside the gem while the runtime owns step execution,
		await suspension, and barrier joins.
	</Card>
	<Card title="Suspend and Resume" icon="document">
		Issue one request with `await.call(...)` or a batch with `await.all(...)`,
		then resume the same node when results are available.
	</Card>
	<Card title="Branch and Join" icon="setting">
		Fan out across multiple edges and let the runtime park branch state until a
		barrier join can release a merged token.
	</Card>
	<Card title="Dynamic Goto" icon="right-arrow">
		Override static edges at runtime with
		[`AsyncGraph::Command.goto`](/async-graph/guides/goto/) and
		[`update_and_goto`](/async-graph/guides/goto/).
	</Card>
</CardGrid>
